# clear_pre.ps1 — Script Analysis, Delivery Methods, and PowerShell Help Metadata

Review of `clear_pre.ps1`, covering design strengths, delivery method compatibility, recommended fixes, and a full guide to adding PowerShell comment-based help and metadata.

Repository: https://github.com/StoicTurk182/IT-Scripts

---

## Script Overview

`clear_pre.ps1` performs selective cleanup of three Windows cache locations:

| Target | Path |
|---|---|
| User Temp | `$env:TEMP` |
| System Temp | `C:\Windows\Temp` |
| Prefetch | `C:\Windows\Prefetch` |

The script reports folder sizes before acting, prompts per-folder confirmation, skips locked files silently, and reports total space reclaimed at completion. Administrator privileges are required.

---

## Design Strengths

### Selective Confirmation Per Target

Rather than clearing everything or nothing, the script prompts individually for each folder. This is significantly safer than bulk-delete tools and reduces the risk of clearing something the user did not intend to.

```powershell
if (Confirm-Action "  Clear $name ($sizeMB MB)?") {
    Clear-Folder -Path $targets[$name]
}
```

### Ordered Dictionary for Target Sequence

Using `[ordered]@{}` ensures the folders are always presented in a consistent, logical order regardless of PowerShell version. A standard hashtable `@{}` does not guarantee insertion order.

```powershell
$targets = [ordered]@{
    "User Temp"   = $env:TEMP
    "System Temp" = "C:\Windows\Temp"
    "Prefetch"    = "C:\Windows\Prefetch"
}
```

### Size Reporting Before and After

Calculating `$totalBefore` and `$totalAfter` and reporting `$reclaimed` gives the operator confirmation that the action had effect — useful when locked files prevent deletion.

### Locked File Handling

`-ErrorAction SilentlyContinue` on `Remove-Item` and `Get-ChildItem` means files held open by running processes are skipped without terminating the script. This is the correct approach — attempting to force-unlock files in use can cause application crashes.

### Single-Responsibility Functions

`Get-FolderSizeMB`, `Clear-Folder`, and `Confirm-Action` are each focused on one task. This makes the script easy to read, test individually, and reuse in other scripts.

## Delivery Method Analysis

### iex (irm "...") Compatibility

This script is **not suitable for `iex` delivery** in its current form for two reasons.

#### Reason 1: `#Requires -RunAsAdministrator`

`iex (irm "...")` inherits the privilege level of the current PowerShell session. It does not elevate. If the session is not already running as administrator, PowerShell throws:

```
The script 'clear_pre.ps1' cannot be run because it contains a "#requires" statement
for running as Administrator.
```

This is not a flaw in the script — admin is legitimately required to write to `C:\Windows\Temp` and `C:\Windows\Prefetch`. The issue is that `iex` cannot satisfy that requirement on its own.

#### Reason 2: `Read-Host` Inside `iex`

`Read-Host` works correctly when a script runs as a `.ps1` file. When the same code runs via `Invoke-Expression`, the interactive prompt behaviour depends on the host environment. In some hosts and pipeline contexts, `Read-Host` inside `iex` returns an empty string silently — meaning every `Confirm-Action` call returns `$false` and nothing is cleared. The script appears to run successfully but takes no action.

Reference: [Invoke-Expression — Microsoft Docs](https://learn.microsoft.com/en-us/powershell/module/microsoft.powershell.utility/invoke-expression)

#### Delivery Method Comparison

| Method | Elevates | Read-Host Safe | Recommended |
|---|---|---|---|
| `iex (irm "...")` from non-admin session | No | Risk | No |
| `iex (irm "...")` from admin session | Already elevated | Risk | No |
| `& ".\clear_pre.ps1"` from admin session | Already elevated | Yes | Yes |
| Module (`Clear-PreCache`) from admin session | Already elevated | Yes | Yes |
| Desktop shortcut with `-Verb RunAs` | Yes (UAC prompt) | Yes | Yes |
| IT-Scripts menu from admin session | Already elevated | Yes | Yes |

### Recommended Delivery: PowerShell Module

The module generated by `Convert-PSScript.ps1` (`Clear-PreCache`) is the correct delivery vehicle. It:

- Runs from an already-elevated session where admin is confirmed
- Preserves `Read-Host` interactivity
- Provides tab completion and `Get-Help` support
- Is version-controlled via the `.psd1` manifest

```powershell
# From an elevated session
Import-Module Clear-PreCache
Clear-PreCache
```

### Recommended Delivery: Desktop Shortcut with Auto-Elevation

For non-technical users who need a double-click experience:

```powershell
$WshShell  = New-Object -ComObject WScript.Shell
$Shortcut  = $WshShell.CreateShortcut("$env:USERPROFILE\Desktop\Clear Cache.lnk")
$Shortcut.TargetPath = "powershell.exe"
$Shortcut.Arguments  = '-NoExit -Command "& ''C:\Path\To\clear_pre.ps1''"'
$Shortcut.Save()

# Set RunAs flag for UAC elevation
$bytes = [System.IO.File]::ReadAllBytes("$env:USERPROFILE\Desktop\Clear Cache.lnk")
$bytes[0x15] = $bytes[0x15] -bor 0x20
[System.IO.File]::WriteAllBytes("$env:USERPROFILE\Desktop\Clear Cache.lnk", $bytes)
```

This produces a shortcut that triggers a UAC prompt on launch, then runs the script in an elevated console.

## Recommended Fixes

### Fix 1: Null Guard on Size Calculation

If `$env:TEMP` is undefined or the path does not exist, `Measure-Object` returns `$null` for `Sum`. Dividing `$null` by `1MB` returns `0` without error in PowerShell, but is misleading — the output shows `0 MB` rather than indicating the path was unavailable.

Current:

```powershell
function Get-FolderSizeMB {
    param ([string]$Path)
    if (-not (Test-Path $Path)) { return 0 }
    (Get-ChildItem $Path -Recurse -Force -ErrorAction SilentlyContinue |
     Measure-Object -Property Length -Sum).Sum / 1MB
}
```

Recommended:

```powershell
function Get-FolderSizeMB {
    param ([string]$Path)
    if (-not (Test-Path $Path)) { return 0 }
    $sum = (Get-ChildItem $Path -Recurse -Force -ErrorAction SilentlyContinue |
            Measure-Object -Property Length -Sum).Sum
    if ($null -eq $sum) { return 0 }
    return $sum / 1MB
}
```

### Fix 2: Add Timestamp to Output

The summary does not record when the cleanup ran. For machines used by multiple people or managed remotely, a timestamp in the output is useful for audit purposes.

Add at the top of the MAIN block:

```powershell
$RunTime = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
Write-Host "`n=== Temp and Prefetch Cleanup === $RunTime`n" -ForegroundColor Cyan
```

### Fix 3: Optional Transcript Logging

For managed environments, logging the output is valuable. Add transcript support matching the pattern used in `Rename-ADUserSmart_v4.ps1`:

```powershell
$LogDir  = "$env:TEMP\CacheCleanLogs"
if (!(Test-Path $LogDir)) { New-Item -Path $LogDir -ItemType Directory -Force | Out-Null }
$LogFile = "$LogDir\CacheClean_$(Get-Date -Format 'yyyyMMdd_HHmm').txt"
Start-Transcript -Path $LogFile -Append | Out-Null
```

At the end:

```powershell
Stop-Transcript | Out-Null
Write-Host "Log saved: $LogFile" -ForegroundColor Gray
```

### Fix 4: Reclaimed Space Edge Case

If nothing is cleared, `$reclaimed` can be a small negative number due to filesystem timing (new temp files written during the script run). Guard against this:

```powershell
$reclaimed = [math]::Round($totalBefore - $totalAfter, 2)
if ($reclaimed -lt 0) { $reclaimed = 0 }
Write-Host ("`n  Reclaimed: $reclaimed MB`n") -ForegroundColor Green
```

## PowerShell Comment-Based Help — Full Guide

PowerShell's comment-based help system allows `Get-Help` to return structured documentation for any function or script. It requires no external tools — the help content is embedded directly in the script as a specially formatted comment block.

Reference: [about_Comment_Based_Help — Microsoft Docs](https://learn.microsoft.com/en-us/powershell/module/microsoft.powershell.core/about/about_comment_based_help)

### Placement Rules

For a script file (`.ps1`), the help block must appear at one of two locations:

- At the very beginning of the file (before any code, including `#Requires`)
- At the very end of the file

For a function, the help block must appear:

- Inside the function body, immediately after the opening `{`
- Or immediately before the `function` keyword

The most common and reliable placement is inside the function body immediately after `{`.

### Full Help Block Structure

```powershell
<#
.SYNOPSIS
    One sentence. What does this script/function do.

.DESCRIPTION
    Full explanation. Can be multiple paragraphs.
    Cover what it does, what it changes, and any important behaviour.

.PARAMETER ParameterName
    Description of this parameter. Repeat this keyword for each parameter.

.PARAMETER AnotherParameter
    Description of this parameter.

.EXAMPLE
    .\clear_pre.ps1
    Runs interactively. Reports sizes and prompts for each folder.

.EXAMPLE
    Import-Module Clear-PreCache
    Clear-PreCache
    Same behaviour via the module.

.INPUTS
    None. This script does not accept pipeline input.

.OUTPUTS
    None. Output is written to the console only.

.NOTES
    Author:   Andrew Jones
    Version:  1.1
    Created:  2026-04-06
    Requires: Administrator privileges
    GitHub:   https://github.com/StoicTurk182/IT-Scripts

.LINK
    https://github.com/StoicTurk182/IT-Scripts

.LINK
    https://learn.microsoft.com/en-us/windows/win32/fileio/temporary-files
#>
```

### Keyword Reference

| Keyword | Purpose | Required |
|---|---|---|
| `.SYNOPSIS` | Single-line summary shown in basic `Get-Help` output | Recommended |
| `.DESCRIPTION` | Full explanation shown with `Get-Help -Full` | Recommended |
| `.PARAMETER` | Documents one parameter — repeat for each | Recommended if params exist |
| `.EXAMPLE` | Shows a usage example — repeat for multiple | Recommended |
| `.INPUTS` | Documents what types the function accepts from the pipeline | Optional |
| `.OUTPUTS` | Documents what types the function returns | Optional |
| `.NOTES` | Free-form — author, version, dates, requirements | Recommended |
| `.LINK` | Related URLs or command names — repeat for multiple | Optional |
| `.COMPONENT` | Categorises the function (e.g. "Disk", "ActiveDirectory") | Optional |
| `.ROLE` | Intended user role (e.g. "Administrator") | Optional |
| `.FUNCTIONALITY` | Describes the intended feature area | Optional |

### Adding Parameter Help

Parameter help is added inside the `<# #>` block using `.PARAMETER` followed by the exact parameter name, then the description on the next line. For `clear_pre.ps1` this is less relevant since it has no parameters, but the pattern for a parameterised version would be:

```powershell
.PARAMETER SkipPrefetch
    If specified, the Prefetch folder is excluded from the target list.
    Useful on systems where Prefetch clearing causes noticeable boot slowdown.

.PARAMETER NoConfirm
    Clears all folders without prompting for confirmation.
    Use with caution — all content in all three folders will be removed.
```

The `.PARAMETER` keyword reads the description from the following lines until the next keyword. The parameter name after `.PARAMETER` must match the variable name in the `param()` block exactly (case-insensitive).

### Version Tracking in .NOTES

A consistent format for `.NOTES` across all scripts in the toolbox makes change history easy to follow:


## Corrected Script: clear_pre_v2.ps1

Applying all fixes and the expanded help block:

```powershell
#Requires -RunAsAdministrator

<#
.SYNOPSIS
    Selectively clears Windows Temp and Prefetch folders with size reporting.

.DESCRIPTION
    Reports the current size of User Temp ($env:TEMP), System Temp
    (C:\Windows\Temp), and the Prefetch cache (C:\Windows\Prefetch).

    Prompts for confirmation before clearing each folder individually.
    Files locked by running processes are skipped silently.
    Total space reclaimed is reported at completion.
    All output is optionally logged to $env:TEMP\CacheCleanLogs\.

.EXAMPLE
    .\clear_pre_v2.ps1
    Reports folder sizes and prompts for confirmation on each target.

.EXAMPLE
    Import-Module Clear-PreCache
    Clear-PreCache
    Runs the same logic via the installed PowerShell module.

.INPUTS
    None.

.OUTPUTS
    None. All output is written to the console host.

.NOTES
    Author:   Andrew Jones
    Version:  1.2
    Created:  2026-04-06
    Modified: 2026-04-07

    Changelog:
      1.2 — Null guard on size calc, timestamp, transcript logging, negative reclaim fix
      1.1 — Initial release

    Requires: Administrator privileges
    GitHub:   https://github.com/StoicTurk182/IT-Scripts

.LINK
    https://github.com/StoicTurk182/IT-Scripts
#>

# ============================================================================
# LOGGING
# ============================================================================

$LogDir  = "$env:TEMP\CacheCleanLogs"
if (!(Test-Path $LogDir)) { New-Item -Path $LogDir -ItemType Directory -Force | Out-Null }
$LogFile = "$LogDir\CacheClean_$(Get-Date -Format 'yyyyMMdd_HHmm').txt"
Start-Transcript -Path $LogFile -Append | Out-Null

# ============================================================================
# FUNCTIONS
# ============================================================================

function Get-FolderSizeMB {
    param ([string]$Path)
    if (-not (Test-Path $Path)) { return 0 }
    $sum = (Get-ChildItem $Path -Recurse -Force -ErrorAction SilentlyContinue |
            Measure-Object -Property Length -Sum).Sum
    if ($null -eq $sum) { return 0 }
    return $sum / 1MB
}

function Clear-Folder {
    param ([string]$Path)
    if (-not (Test-Path $Path)) {
        Write-Host "  Path not found: $Path" -ForegroundColor Yellow
        return
    }
    Remove-Item "$Path\*" -Recurse -Force -ErrorAction SilentlyContinue
}

function Confirm-Action {
    param ([string]$Prompt)
    $response = Read-Host "$Prompt [Y/N]"
    return $response -match '^[Yy]$'
}

# ============================================================================
# MAIN
# ============================================================================

$RunTime = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
Write-Host "`n=== Temp and Prefetch Cleanup === $RunTime`n" -ForegroundColor Cyan

$targets = [ordered]@{
    "User Temp"   = $env:TEMP
    "System Temp" = "C:\Windows\Temp"
    "Prefetch"    = "C:\Windows\Prefetch"
}

# --- Report sizes ---
Write-Host "Current folder sizes:`n" -ForegroundColor Yellow

$sizes      = @{}
$totalBefore = 0

foreach ($name in $targets.Keys) {
    $size          = Get-FolderSizeMB -Path $targets[$name]
    $sizes[$name]  = $size
    $totalBefore  += $size
    Write-Host ("  {0,-15} {1,8} MB" -f $name, [math]::Round($size, 2))
}

Write-Host ("  {0,-15} {1,8} MB`n" -f "TOTAL", [math]::Round($totalBefore, 2)) -ForegroundColor White

# --- Prompt and clear ---
Write-Host "Select folders to clear:`n" -ForegroundColor Yellow

$cleared = @()

foreach ($name in $targets.Keys) {
    $sizeMB = [math]::Round($sizes[$name], 2)
    if (Confirm-Action "  Clear $name ($sizeMB MB)?") {
        Clear-Folder -Path $targets[$name]
        $cleared += $name
        Write-Host "  $name cleared.`n" -ForegroundColor Green
    } else {
        Write-Host "  $name skipped.`n" -ForegroundColor DarkGray
    }
}

# --- Summary ---
Write-Host "Summary:" -ForegroundColor Yellow

$totalAfter = 0
foreach ($name in $targets.Keys) {
    $size        = Get-FolderSizeMB -Path $targets[$name]
    $totalAfter += $size
    $status      = if ($cleared -contains $name) { "cleared" } else { "skipped" }
    Write-Host ("  {0,-15} {1,8} MB  ({2})" -f $name, [math]::Round($size, 2), $status)
}

$reclaimed = [math]::Round($totalBefore - $totalAfter, 2)
if ($reclaimed -lt 0) { $reclaimed = 0 }
Write-Host ("`n  Reclaimed: $reclaimed MB`n") -ForegroundColor Green

# ============================================================================
# CLEANUP
# ============================================================================

Stop-Transcript | Out-Null
Write-Host "Log saved: $LogFile`n" -ForegroundColor Gray
```
