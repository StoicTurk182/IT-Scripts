# SOP: Static WiFi QR Code Generation

Generate a permanent, self-contained WiFi QR code that does not rely on any third-party service or expiring URL.

## Problem Statement

QR codes generated by online tools often stop working after a short period. This is because many online QR generators produce dynamic QR codes that encode a redirect URL to their server rather than encoding the WiFi credentials directly. When the free link expires or the service revokes access, the QR code breaks even though the WiFi password has not changed.

The fix is to generate a static QR code where the WiFi connection string is encoded directly into the QR image. Static QR codes have no dependency on any external service and do not expire.

## How to Identify the Problem

Scan the broken QR code with a phone camera:

| Result | Type | Cause |
|--------|------|-------|
| Phone opens a URL in a browser | Dynamic QR | The QR points to a third-party server that has expired or revoked the link |
| Phone prompts to join a WiFi network | Static QR | The QR is correct; the issue is elsewhere (password changed, SSID renamed) |
| Phone does nothing | Damaged QR | Print quality too low, image too small, or QR is corrupt |

If the result is a URL, the QR must be regenerated as a static code using the method below.

## Prerequisites

- The SSID (network name) of the visitor WiFi, exact spelling, case-sensitive
- The current WiFi password
- A computer with one of: Python installed, PowerShell 5.1+, or access to a trusted offline QR generator

## WiFi QR String Format

The string to encode follows the ZXing WiFi URI specification:

```
WIFI:T:WPA;S:<SSID>;P:<Password>;H:<Hidden>;;
```

| Field | Value | Notes |
|-------|-------|-------|
| T | WPA | Use `WPA` for WPA2 and WPA3 networks |
| S | SSID | Exact network name, case-sensitive |
| P | Password | Plaintext passphrase |
| H | false | Set to `true` only if the SSID is hidden |

The trailing double semicolon `;;` is required.

### Example

SSID: `OfficeGuest`
Password: `Welcome2025!`

```
WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;
```

## Step-by-Step: Generate with Python

This is the recommended method. Python produces the QR image locally with no internet dependency.

### Step 1: Install the qrcode Library

```bash
pip install qrcode[pil]
```

### Step 2: Build the WiFi String

Replace the SSID and password with the actual values:

```
WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;
```

### Step 3: Generate the QR Image

Command line (one-liner):

```bash
qr "WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;" > wifi-qr.png
```

Or as a script for more control over size and output:

```python
import qrcode

wifi_string = "WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;"

qr = qrcode.QRCode(
    version=1,
    error_correction=qrcode.constants.ERROR_CORRECT_H,
    box_size=20,
    border=4,
)
qr.add_data(wifi_string)
qr.make(fit=True)

img = qr.make_image(fill_color="black", back_color="white")
img.save("wifi-qr.png")

print("QR code saved to wifi-qr.png")
```

Save the above as `generate-wifi-qr.py` and run:

```bash
python generate-wifi-qr.py
```

### Step 4: Verify the QR Code

Before printing, scan the generated image with a phone camera and confirm:

1. The phone prompts to join a WiFi network (not open a URL)
2. The SSID shown matches the intended network
3. Tapping "Join" or "Connect" successfully connects to the network

If the phone opens a browser URL instead, the QR was not generated correctly. Re-check the WiFi string.

## Step-by-Step: Generate with PowerShell

### Step 1: Install the QRCodeGenerator Module

```powershell
Install-Module -Name QRCodeGenerator -Scope CurrentUser
```

### Step 2: Generate the QR Image

```powershell
New-QRCodeURI -URI "WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;" -OutPath "C:\Temp\wifi-qr.png"
```

### Step 3: Verify

Open `C:\Temp\wifi-qr.png` and scan with a phone. Confirm it prompts to join WiFi, not open a URL.

## Step-by-Step: Generate Without Python or PowerShell

If neither Python nor PowerShell with the QRCodeGenerator module is available:

1. Open a trusted offline-capable QR tool. Options that work locally without sending data to a server:
   - **LibreOffice Writer** — Insert > OLE Object > QR Code (built-in, no internet required)
   - **Firefox/Chrome** — the `qr-code` Web Component can generate client-side in the browser with no server call
2. Paste the full WiFi string: `WIFI:T:WPA;S:OfficeGuest;P:Welcome2025!;H:false;;`
3. Generate and export as PNG
4. Scan to verify it prompts WiFi join, not a URL

Do not use an online QR generator unless you are certain it produces static codes and you accept the risk of exposing the WiFi password to a third party.

## Printing Guidelines

| Setting | Recommendation |
|---------|---------------|
| Minimum QR size | 3cm x 3cm (approximately 1.2 inches) for reliable scanning at arm's length |
| Colour | Black QR on white background, maximum contrast |
| Lamination | Recommended for durability in reception areas |
| Fallback text | Print the SSID and password in plain text below the QR for devices without cameras |

## Verification Checklist

After generating and before distributing:

| Check | Expected Result | Pass |
|-------|----------------|------|
| Scan QR with iPhone Camera | Prompts "Join OfficeGuest?" | Y/N |
| Scan QR with Android Camera | Prompts to connect to WiFi | Y/N |
| Tap Join/Connect | Device connects to the network | Y/N |
| QR does not open a browser URL | No URL shown, only WiFi prompt | Y/N |
| SSID displayed matches intended network | Exact match including case | Y/N |

If any check fails, regenerate the QR and verify the WiFi string matches the SSID and password exactly.

## If the QR Stops Working in Future

| Symptom | Cause | Action |
|---------|-------|--------|
| QR scans but password rejected | WiFi password has changed | Regenerate QR with new password |
| QR scans but network not found | SSID renamed or disabled | Confirm current SSID and regenerate |
| QR opens a URL | Dynamic QR was used by mistake | Regenerate using this SOP |
| QR does not scan | Print faded or too small | Reprint at minimum 3cm x 3cm |

## Common Mistakes

| Mistake | Why It Breaks | Fix |
|---------|--------------|-----|
| Using an online QR generator | Many produce dynamic redirect QRs that expire | Use offline generation (Python, PowerShell, LibreOffice) |
| SSID typo or wrong case | WiFi string does not match actual network name | Copy SSID exactly from wireless settings |
| Missing trailing `;;` | QR may not be parsed correctly by all devices | Always end the string with `;;` |
| Encoding as plain text instead of WiFi URI | Phone shows the password as text instead of prompting to join | String must start with `WIFI:T:` |
| Printing too small | Camera cannot resolve the QR pattern | Minimum 3cm x 3cm |
